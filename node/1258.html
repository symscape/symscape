Caedium 4.0.2 Release Notes

Submitted by symscape on July 31, 2012 - 18:14
  • Release Notes

The latest Caedium release has a broad range of new features, enhancements, and bug fixes. It includes an array of new flow solver options for multiphase, moving reference frames, and porous media. It has new boundary conditions for fans and cyclics. It also includes internationalization support for a selection of popular languages.

New Features and Enhancements

  • More Physics
    • Simulate air and water (or any other immiscible multiphase fluids) with the Volume Of Fluid (VOF) solver [RANS Flow]
    • Simulate rotating machinery (e.g., mixers) with Moving Reference Frames (MRF) [RANS Flow]
    • Simulate porous volumes and faces [RANS Flow]
    • Simulate fans at boundaries and embedded within a flow domain [RANS Flow]
    • Reduce simulation time and memory requirements with the cyclic boundary condition [RANS Flow]
    • Model boundary layer transitions with the kt-kl-omega low-Reynolds-number turbulence model [RANS Flow]
    • Perform more robust simulations now that the k-omega SST is the default turbulence model [RANS Flow]
  • Improved Performance
    • Turn simulations around faster with more aggressive default parallel preferences [RANS Flow]
    • Run larger simulations on Mac OS X with a 64-bit version of Caedium
    • Use Windows HPC Scheduler services in Azure without an on-premise installation of Windows HPC Server [RANS Flow]
  • Better Visualization
    • Create stunning visuals with an extensive array of animated particles that includes arrows and cones [RANS Flow|Panel Flow]
    • Visualize and report wall heat flux and rho when supported by a solver [RANS Flow]
  • Miscellaneous
    • Enjoy native language translations of Caedium in English, French, German, Italian, Japanese, Portuguese (Brazil), and Spanish
    • Exchange membrane flow results with ixForten 4000 [RANS Flow|Panel Flow]
    • Import facets more reliably with a node merging tolerance preference [Exchange]

Bug Fixes

  • During multiple selection (holding down Ctrl) sometimes selection remained highlighted when unselected
  • After transforming a cylinder, subtracting from a face (topology only), and stitching into a volume, faces were transformed twice [Builder]
  • ABL profile was wrong [RANS Flow]
  • Same face remeshed indefinitely after requesting Area color map [RANS Flow]
  • Tetrahedral meshing of 2-sided faces unreliable when in close proximity to other geometry [RANS Flow]
  • Two-sided face construction failed for face with a seam, e.g., cylinder [Builder]
  • Creating 2-sided face between connected volumes did not preserve connected faces [Builder]
  • After WRL export switching view caused crash [Viz Export]
  • Restoring compressible cases produced warnings: Unknown Property: 'alphat' [RANS Flow]
  • Edge parameter out of bounds in OpenCascade caused crash [Builder]
  • When tetrahedral meshing failed immediately and RANS Flow solver tried to run it caused a crash [RANS Flow]
  • IGES import and shade for a specific model caused a crash [Exchange]
  • Loft through 2 circles caused crash [Builder]
  • Stitching a volume (topology only) for a specific model produced incorrect solid orientation [Builder]
  • Subtracting (topology only) edge caused crash [Builder]
  • Subtracting a faceted face from a volume caused crash [Builder]
  • Could not copy and paste into Residual Monitor for a volume group [RANS Flow]
  • Adjustable time step failed to extract results [RANS Flow]
  • Export OBJ with no points caused crash [Viz Export]
  • TFI meshing for a specific model caused crash [RANS Flow]
  • Shutdown while plot updating caused crash
  • Hidden properties sometimes visible
  • FunctionObjects re-extracted unnecessarily [RANS Flow]
  • Clipping large cell count models with complex visualization caused crash [RANS Flow|Panel Flow]
  • Interpolating results onto non-flow-domain geometry caused crash [RANS Flow]
