Caedium Version 2.3.0 Release Notes

Submitted by symscape on September 7, 2010 - 13:52
  • Release Notes

The OpenFOAM®1 solvers and utilities used by the Caedium RANS Flow and Professional add-ons have been upgraded to OpenFOAM 1.7.x from 1.6.x. Also the unsteady incompressible flow solvers in Caedium now use PIMPLE instead of PISO. PIMPLE has better stability for large time steps compared to PISO.

New Features and Enhancements

  • Solvers and utilities upgraded to OpenFOAM 1.7.x from 1.6.x [RANS Flow]
  • New Unsteady Incompressible Solvers
    • pimpleFoam replaces pisoFoam as the unsteady incompressible flow solver [RANS Flow]
    • buoyantBoussinesqPimpleFoam replaces buoyantBoussinesqPisoFoam as the unsteady incompressible flow solver with buoyancy and heat transfer [RANS Flow]
  • Continuous wall functions are now the default for high-Reynolds-number (wall function) turbulence models. Continuous wall functions provide better accuracy over standard wall functions. [RANS Flow]
  • The default media player is launched after writing an animation file (.avi) on Windows and Mac [RANS Flow|Transient]

Bug Fixes

  • nan was not recognized as a RANS Flow solver singularity (failure) on Windows [RANS Flow]
  • Changing simulation boundary names midway through a RANS Flow simulation caused an error [RANS Flow]
  • Could not over-write an animation file (.avi) on Windows [RANS Flow|Transient]
  • View window was missing a redraw after camera transform while performing an animation [RANS Flow|Transient]
  • TwitPic failed to broadcast any images [RANS Flow|Transient]
  • Switching the Panel Flow state from steady to unsteady (transient) midway through a simulation caused Caedium to crash [Panel Flow]
  • The space character in a temporary directory name prevented a RANS Flow simulation from starting on Linux and Mac [RANS Flow]
